# Production Bug — Audio Works on Android but Not on iPhone/Safari

The audio experience works on Android, but does NOT work on iPhone/Safari.

This must be debugged specifically for iOS Safari.

## Current behavior

* Android: audio works
* Desktop/dev: audio works
* iPhone/Safari: user clicks Enable Sound, but no audio is heard

## Required debugging

Check iOS-specific restrictions:

1. Is playback triggered directly inside the user click handler?
2. Is the YouTube/player/audio element already mounted before the click?
3. Is the player muted or volume blocked on iOS?
4. Does iOS block hidden iframe playback?
5. Does iOS require the player to be visible / non-zero size?
6. Is the play promise rejected on iOS?
7. Are there Safari console errors?

## Important iOS requirement

For iPhone/Safari, playback must be triggered synchronously from the actual user gesture.

Do not defer playback through:

* setTimeout
* async state updates
* delayed callbacks
* promise chains after the click

The click handler must directly call the player/audio play action.

## Required fix

Make the audio experience work on iPhone/Safari after the user explicitly taps Enable Sound.

If YouTube hidden playback is blocked by iOS Safari, report that clearly and provide the safest fallback.

Do NOT break Android.
Do NOT break desktop.
Do NOT change design.
Do NOT remove the floating mute/unmute button.

## Deliverable

Send:

1. root cause on iPhone
2. exact files changed
3. whether YouTube hidden playback is supported on iOS
4. what fallback was implemented if needed
5. confirmation that Android still works
6. confirmation that iPhone/Safari works after user tap

This must be done in a structured, scalable, and premium way.

---

## 1. Multiple Phone Numbers (Enhanced)

The system must support up to **5 phone numbers**.

Each phone entry must include:

* phone number
* custom free-text label

### Examples of labels:

* Toll-free number
* Calls from Israel
* International calls
* VIP desk
* Direct line

---

## 2. Multiple Email Addresses (NEW)

The system must also support up to **5 email addresses**.

Each email entry must include:

* email address
* custom free-text label

### Examples of labels:

* General inquiries
* VIP requests
* Sales
* Support
* Medical flights

The label must be fully editable (free text).

---

## 3. Dynamic behavior (critical)

For BOTH phones and emails:

* If only 1 entry is filled → show only 1
* If 2 are filled → show 2
* If 3 → show 3
* and so on

No empty placeholders.
No fake or unused fields shown.

Everything must be fully dynamic and admin-controlled.

We must NOT overload the UI.

### Primary areas (header / CTA / floating / main buttons)

In key high-visibility areas:

* show ONLY ONE primary phone number
* clean and minimal
* no clutter

This should remain consistent with current premium design.

---

### Full contact display areas

In areas where users expect full details:

* Contact page
* Footer (if appropriate)
* Dedicated contact sections

* ALL phone numbers (with labels)
* ALL email addresses (with labels)

Label + Value

Example:

* Toll-free number — +1 800 123 4567

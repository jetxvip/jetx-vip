Please update the accessibility button implementation on the website as follows:

1. REMOVE ONE ACCESSIBILITY BUTTON
   There should be only ONE accessibility button on the site.
   If two accessibility buttons are currently appearing, remove the duplicate and leave only a single accessibility button active.

2. MATCH THE BUTTON SIZE TO THE WHATSAPP ICON
   Adjust the accessibility button icon and button dimensions so they match the size and visual scale of the existing WhatsApp icon currently shown on the website.

Important:

* The accessibility button should feel visually consistent with the WhatsApp button
* Match the icon size, button size, spacing, and floating presence as closely as possible
* Keep it elegant, minimal, and premium
* Do not make it larger than the WhatsApp button
* Do not make it visually dominant

3. KEEP PROPER LANGUAGE BEHAVIOR
   The accessibility panel must still match the active language of the site:

* If the site is in Hebrew mode, show Hebrew only and RTL layout
* If the site is in English mode, show English only and LTR layout

4. EXPAND THE ACCESSIBILITY PANEL FEATURES
   Please significantly expand the available functions inside the accessibility button panel.

Current panel is too limited.
Add a broader but still clean and usable accessibility tools set.

Required features to add:

TEXT / READABILITY:

* Increase text size
* Decrease text size
* Reset text size
* Increase line spacing
* Increase letter spacing
* Readable font toggle
* Dyslexia-friendly font toggle (if possible)

VISUAL ACCESSIBILITY:

* High contrast mode
* Invert colors
* Brightness reduction / dark reading mode
* Grayscale mode
* Highlight links
* Highlight headings
* Bigger cursor
* Reading guide / reading ruler (if possible)

MOTION / FOCUS:

* Reduce animations / motion
* Pause flashing or moving effects if any exist
* Focus highlight for interactive elements

STRUCTURE / ORIENTATION:

* Highlight buttons
* Highlight forms
* Highlight all clickable elements
* Page structure aid if possible

RESET:

* Reset all accessibility settings button must remain available and clear

5. PANEL DESIGN REQUIREMENTS
   Even with more features, keep the panel:

* clean
* well organized
* not overloaded visually
* grouped by category
* easy to use on mobile and desktop
* premium and minimal in appearance

Recommended grouping:

* Text
* Visual
* Navigation / Focus
* Reset

6. PANEL UX REQUIREMENTS

* Must remain lightweight
* Must not break layout
* Must not overlap major CTA buttons unnecessarily
* Must remain keyboard accessible
* Must remain mobile friendly
* Must not slow down the site significantly

7. PERSISTENCE
   Accessibility settings may persist using localStorage.
   Reset must fully clear all saved accessibility preferences.

8. FINAL RESULT REQUIRED
   After this update:

* only one accessibility button should remain

* the experience should remain elegant, simple, and premium

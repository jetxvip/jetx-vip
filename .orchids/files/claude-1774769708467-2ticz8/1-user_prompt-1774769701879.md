JetX.vp — Add “Sold Out” State for Empty Legs and Special Offers

I want to add a new feature for both:

* Empty Leg flights
* Special Offers

## Goal

I want to be able to keep sold items visible on the site, but clearly mark them as no longer available.

Instead of deleting them, I want an admin-controlled “Sold Out” state.

---

## Required behavior

For every Empty Leg flight and every Special Offer, add an option in the admin panel to mark the item as:

* Sold Out

When an item is marked as Sold Out:

### On the public site:

* the item must remain visible
* it must show a clear red line / sold-out overlay treatment
* it must display the words:

  * **Sold Out**

This should make it visually obvious that:

* the offer/flight existed
* it is no longer available

---

## Design requirements

When an item is marked as Sold Out:

* add a clear red visual treatment
* add a visible “Sold Out” label
* make sure it is elegant and premium, not cheap-looking
* the card should still look good and remain readable
* the sold-out state must be immediately noticeable

Possible treatment:

* diagonal red line or red strike treatment
* sold-out badge
* reduced CTA activity

---

## CTA behavior

For sold out items:

* the normal booking/request CTA must NOT behave like an available offer
* either:

  * disable the CTA
  * or replace it with a non-booking message/button such as:

    * Sold Out
    * No Longer Available

But users must not be able to submit a booking request for a sold-out item as if it were still active.

---

## Admin requirements

In the admin panel, for each:

* Empty Leg flight
* Special Offer

Add a simple toggle / checkbox / status control:

* Available
* Sold Out

This status must be editable anytime.

---

## Data / persistence requirements

The Sold Out state must:

* be saved in persistent storage
* sync across devices
* remain after refresh
* appear correctly in production

---

## Important

Do NOT delete sold items automatically.

The purpose is to keep them visible on the site for trust, demand proof, and marketing value — while clearly showing they are no longer available.

---

## Final requirement

This feature must work for:

* existing items
* newly created items
* both languages
* all responsive breakpoints

Please implement this cleanly without breaking the current admin flow or the existing card design.

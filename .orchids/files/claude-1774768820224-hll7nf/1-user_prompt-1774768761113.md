JetX.vp — Empty Leg Flight Direction Bug (RTL Logic Incorrect)

The flight direction indicator in Empty Leg cards is still incorrect in Hebrew (RTL mode).

This is NOT a styling issue — it is a logic issue.

---

## CURRENT BUG

In Hebrew:

* The plane icon is pointing / positioned from destination back to origin
* The visual direction is reversed
* It creates the impression that the flight goes in the wrong direction

This is incorrect and confusing.

---

## ROOT PROBLEM

The UI appears to be mirrored for RTL,
BUT the flight direction itself is NOT being preserved.

You are flipping layout — but also unintentionally flipping meaning.

---

## REQUIRED FIX (STRICT)

The flight direction must ALWAYS represent:

ORIGIN → DESTINATION

regardless of language or layout direction.

---

## CORRECT BEHAVIOR

### English (LTR)

* left → right
* origin on the left
* destination on the right
* plane points toward destination

### Hebrew (RTL)

* layout is RTL
  BUT:

* origin and destination labels may switch sides visually

* HOWEVER the plane must STILL point toward the destination

The direction of the plane must be based on DATA, not layout.

---

## IMPORTANT

Do NOT:

* blindly flip the entire component
* reverse the direction logic

Instead:

* keep the logical direction (origin → destination)
* adjust positioning without breaking meaning

---

## EXPECTED RESULT

For ANY language:

* the plane icon must always indicate movement toward the destination
* the route must be immediately understandable
* no case should visually suggest reverse direction

---

## DEBUG HINT

Check where the direction is determined:

If you are using something like:

* flex-direction
* transform: scaleX(-1)
* RTL flipping

Make sure this does NOT invert the semantic direction.

The arrow/plane direction must be explicitly controlled.

---

## FINAL NOTE

This is a critical UX bug.

Users must instantly understand the flight direction without confusion.

Fix the logic — not just the layout.

# iPhone/Safari Bug — Hero YouTube Video Not Loading Properly

The audio experience now works, but there is a separate issue with the visible airplane/hero YouTube video.

## Current behavior

* Android: hero airplane YouTube video loads correctly
* Windows desktop: video loads correctly
* iPhone/Safari: video does NOT load properly
* On iPhone, only the YouTube icon/logo appears instead of the actual video

This happens when testing from iPhone, including in Cyprus.
Sometimes on desktop the YouTube logo appears briefly and then the video loads, but on iPhone it stays stuck.

---

## Required debugging

Please inspect the visible hero/background YouTube video embed.

Check specifically for iOS Safari requirements:

1. The iframe/player must include:

   * `playsinline=1`
   * `mute=1`
   * `autoplay=1`
   * `controls=0`
   * `rel=0`
   * `modestbranding=1`
   * `enablejsapi=1` if JS control is used

2. The video must be muted before autoplay starts.

3. The iframe must not be `display: none`, zero-size, or visually hidden.

4. Check if lazy loading delays the YouTube iframe incorrectly on iOS.

5. Check whether the video container has proper width/height on iPhone.

6. Check if CSS overlays, z-index, opacity, or pointer-events are covering the iframe on iOS.

7. Check if CSP allows all required YouTube assets in production:

   * `https://www.youtube.com`
   * `https://www.youtube-nocookie.com`
   * `https://s.ytimg.com`
   * `https://*.googlevideo.com`

8. Check Safari console/network if possible for blocked YouTube/video requests.

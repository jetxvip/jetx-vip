GOAL:

TASKS:

1. Introduce HttpOnly Secure Cookie-based authentication (progressive migration)
- Keep existing Bearer token support for backward compatibility
- On login:
  - Issue JWT as BOTH:
    a) HttpOnly Secure SameSite=Strict cookie
    b) (optional) response body for legacy compatibility
- Update server endpoints to accept auth from:
  - Authorization header (existing)
  - OR cookie (new)
- Do NOT break existing frontend behavior
- Gradually prefer cookie over localStorage

2. Secure Cookie Configuration
- HttpOnly: true
- Secure: true (production)
- SameSite: Strict
- Path: '/'
- Proper expiration aligned with token TTL

3. Token Strategy Improvements
- Keep JWT but:
  - Reduce TTL further (e.g. 1–4 hours)
  - Add "issued at" validation
  - Enforce expiration strictly
- Keep denylist logic
- Ensure logout invalidates cookie AND denylists token

4. KV-based persistent rate limiting (upgrade current system)
- Replace in-memory limiter with KV-backed limiter
- Track by IP (and optionally email)
- Sliding window (e.g. 5 attempts / 15 min)
- Lockout stored in KV
- Prevent reset on cold start

5. Admin Login Hardening
- Normalize login responses (no info leaks)
- Add slight artificial delay on failure (anti-brute force)
- Ensure constant-time comparisons remain

6. CSP Hardening (safe mode)
- Keep existing CSP but:
  - Audit actual used domains
  - Allow only required sources
  - Avoid breaking fonts/images/API calls
- Add report-only mode fallback if needed

7. Advanced Security Headers
Ensure all responses include:
- Strict-Transport-Security (already exists, validate config)
- Content-Security-Policy (tightened)
- X-Frame-Options: DENY
- X-Content-Type-Options: nosniff
- Referrer-Policy
- Permissions-Policy
- Cross-Origin-Opener-Policy
- Cross-Origin-Resource-Policy

8. CORS Hardening (final pass)
- Ensure NO endpoint uses '*'
- Allow only:
  - production domain (jetx.vip + www)
  - vercel preview domains
  - localhost (dev only)

9. Auth Middleware Unification
- Create shared auth utility:
  - extracts token from header OR cookie
  - verifies JWT
  - checks denylist
- Reuse across ALL protected endpoints

10. Logging Improvements
- Structured logs:
  - login attempts
  - failures
  - lockouts
- NEVER log secrets or passwords
- Include IP + timestamp

11. Prepare 2FA (do NOT enforce yet)
- Add optional hook point in login flow
- Add placeholder logic for OTP verification
- Do NOT require external provider yet

12. Security Sanity Checks
- Ensure no secrets leak to client bundle
- Ensure no debug logs in production
- Ensure no sensitive data in responses

JetX.vp — Notification System Not Connected to Leads (Critical Fix)

The notification bell UI exists, but it is NOT connected to the actual lead system.

Current behavior:

* new lead appears in the Leads list ✔️
* BUT:

  * bell does NOT update ❌
  * unread indicator does NOT appear ❌
  * sound does NOT play ❌

This means the notification system is not wired to lead events.

---

## REQUIRED FIX

The notification system must be fully connected to lead creation events.

---

## 1. Trigger on New Lead

Whenever a new lead is created (from ANY source):

* Quote
* Empty Leg
* Offer

The system must trigger a notification event.

---

## 2. Bell Update

On new lead:

* increment unread count
* show badge (number or dot)
* visually update the bell

---

## 3. Sound Trigger

On new lead:

* play notification sound (chime)
* must work when admin panel is open
* must not depend on refresh

---

## 4. Real-Time Behavior

When admin panel is open:

* new lead arrives → immediate reaction:

  * toast ✔️ (already exists)
  * bell update (missing)
  * sound (missing)

No page refresh should be required.

---

## 5. Data Source Consistency

The notification system must listen to the SAME data source as the leads list.

If leads are coming from:

* KV
* API
* polling / fetch

Then notifications must be triggered from the same source.

Right now, the bell is disconnected from that flow.

---

## 6. Unread Logic

* new leads must be marked as "unread"
* unread count increases
* when admin views leads → mark as read → reset counter

---

## 7. Do NOT fake this

This must NOT be:

* UI-only
* hardcoded
* triggered manually

It must be tied to real lead creation events.

---

## EXPECTED RESULT

When a new lead is submitted:

1. It appears in Leads ✔️
2. Toast appears ✔️
3. Bell updates ❗
4. Unread badge appears ❗
5. Sound plays ❗

All of this must happen instantly, without refresh.

---

Currently, steps 3–5 are not working and must be fixed.

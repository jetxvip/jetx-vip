יש כמה בעיות קריטיות שצריך לתקן במערכת האדמין והאימות לפני שנמשיך:

Login inputs must always be empty
שדות האימייל והסיסמה במסך /login חייבים להיות ריקים תמיד.
אין להציג default values.
אין להציג example credentials בתוך השדות.
אין צורך גם ב-placeholder שמרמז על שם משתמש או סיסמה אמיתיים.
המשתמש שצריך להתחבר כבר יודע מה להכניס.
No credentials should remain visible in the login form
כרגע נראה שהפרטים נשארים מוצגים במסך ההתחברות.
יש לוודא שלא נשמרים credentials בתוך ה־state, localStorage, autofill פנימי, hardcoded values או default form values.
השדות צריכים להיטען ריקים בכל פתיחה של /login.
Session timeout after 10 minutes of inactivity
כרגע אחרי login נשארים מחוברים עד logout ידני בלבד.
זה לא תקין.
נדרש inactivity timeout של 10 דקות.
אם אין פעילות במשך 10 דקות, הסשן צריך לפוג אוטומטית.
לאחר timeout:
למחוק token/session
לנתב ל־/login
להציג הודעה ברורה כמו:
"Your session expired due to inactivity. Please sign in again."
Activity-based session handling
יש לאפס את הטיימר בכל פעילות משתמש רלוונטית:
mousemove
keydown
click
scroll
touchstart
ה־timeout צריך להיות לפי inactivity, לא רק לפי זמן מאז login.
Admin route protection must remain strict
/admin חייב להישאר protected route.
בלי token תקף אין גישה.
אם token expired או invalid:
למחוק auth state
להפנות ל־/login
Logout and View Site buttons are not visible enough
בחלק העליון של ה־admin panel כפתורי Logout ו־View Site לא קריאים מספיק.
צריך לתקן contrast / colors / hover / focus states כך שהם יהיו ברורים וקריאים על כל רקע.
זה תיקון UI חשוב, לא לשנות את מבנה המערכת אלא רק לשפר visibility ונגישות.
Harden admin/auth security without breaking existing functionality
Please harden the system carefully, without damaging anything that already works.

Required hardening:

no hardcoded credentials anywhere
no fallback credentials
token validation on protected routes
token expiration handling
automatic logout on inactivity
clear auth cleanup on logout and expiry
prevent stale session reuse
avoid exposing sensitive auth data in client storage unnecessarily
keep existing login/admin/storage flows working
Do not break existing admin functionality
כל התיקונים צריכים לשמור על:
login
logout
protected admin
save/update flows
dynamic content management
contact values updates across the site

Please implement these fixes in the actual project code and return a deployment-ready version.

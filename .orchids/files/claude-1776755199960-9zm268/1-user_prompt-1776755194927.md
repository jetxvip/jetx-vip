## Goal

We need to convert the current website from a React/Vite SPA into an SEO-friendly architecture that is suitable for strong Google indexing.

The current project behaves like a SPA with routing through `index.html`, which is not ideal for organic search performance on a luxury aviation website. The goal is to migrate the marketing site to **Next.js** with proper **SSR / SSG**, while preserving the existing branding, content, multilingual system, and admin/API behavior.

---

## Critical Requirements

Do NOT:

* redesign the site
* change branding
* rewrite copy unless technically required
* break the admin flow
* break current lead/contact/quote flows
* remove existing languages (`en`, `he`, `ar`, `ru`)
* change the Russian tone we already finalized

Do:

* migrate the frontend architecture to **Next.js**
* make each major page a real route with indexable HTML
* preserve current UI/UX as closely as possible
* preserve current API behavior or migrate it safely to Next.js API routes / Vercel-compatible server functions

---

The current project appears to be:

## Required Technical Migration

### 1. Move the site to Next.js

Use Next.js as the main framework for production deployment.

Preferred approach:

* App Router or Pages Router is acceptable
* choose the approach that is safest and clearest for SEO
* preserve the site structure and visual behavior

### 2. Convert marketing pages into real SEO pages

Create real routes for the main public pages, including at minimum:

* `/`
* `/about`
* `/services`
* `/fleet`
* `/empty-legs`
* `/contact`
* `/quote`
* `/ambulance`
* aircraft detail pages as real dynamic routes if applicable

Each page must return meaningful HTML on first load, not content hidden behind client-only rendering.

### 3. Use SSR / SSG correctly

Use:

* **SSG** for mostly static marketing pages
* **SSR** only where needed for dynamic content
* choose whichever is most appropriate page by page

Preferred:

* Home, About, Services, Contact: static generation
* Fleet / aircraft detail / empty legs: static generation or server rendering depending on current data flow
* Admin can remain client-heavy if needed, but public marketing pages must be SEO-first

### 4. Preserve multilingual structure

Keep existing languages:

* `en`
* `he`
* `ar`
* `ru`

Requirements:

* language switching must still work
* page HTML must reflect the selected language
* set proper `<html lang="">`
* preserve RTL/LTR handling
* preserve the existing Russian premium tone exactly

If possible, implement SEO-friendly localized routes or a robust language-aware routing structure.

### 5. Metadata for every page

Every public page must have:

* unique `<title>`
* unique meta description
* canonical URL
* Open Graph tags
* Twitter tags
* correct `lang`
* proper heading hierarchy (`h1`, `h2`, etc.)

### 6. Structured data

Add relevant schema markup where appropriate:

* Organization
* Website
* LocalBusiness or ProfessionalService if applicable
* FAQ schema where justified
* Breadcrumb schema where applicable

### 7. Sitemap and robots

Generate:

* `sitemap.xml`
* `robots.txt`

Ensure all important public pages are included and indexable.

### 8. Preserve or safely migrate backend/API behavior

Current Vercel-compatible API behavior must remain functional.

If the current project uses:

* `api/auth.js`
* `api/store.js`
* `api/verify.js`

then either:

* preserve them in a Vercel-compatible structure
* or migrate them cleanly to Next.js API routes / route handlers

Do NOT break:

* admin login
* KV sync
* lead saving
* quote/contact form submissions

### 9. Performance and crawlability

Ensure:

* meaningful HTML is present in the initial response
* no critical marketing content depends only on client-side JS to appear
* pages are crawlable by Google
* internal links are standard crawlable links
* no accidental blocking in robots or meta tags

### 10. Deliverables

Provide:

1. migrated Next.js project structure
2. public SEO-friendly pages
3. working multilingual support
4. working forms and APIs
5. deployment-ready configuration for Vercel
6. short deployment notes for a non-technical site owner

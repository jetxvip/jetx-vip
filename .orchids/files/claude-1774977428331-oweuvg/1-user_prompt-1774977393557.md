ORCHID IMPLEMENTATION PROMPT – TERMS OF USE PAGE (MULTI-LANGUAGE)

Build a dedicated “Terms of Use” page for the website with full support for Hebrew and English, including proper language switching behavior.

GOAL:
Create a clean, professional, premium-looking Terms of Use page that dynamically changes content based on the current site language.

LANGUAGE REQUIREMENT (CRITICAL):

The website has two modes only:

1. Hebrew
2. English

The Terms of Use page must fully adapt to the current site language.

If the site is in Hebrew mode:

* Display ONLY Hebrew content
* Display:

  1. Terms of Use – Israel (Hebrew)
  2. Immediately after: Terms of Use – Europe (Hebrew)
* Layout direction: RTL
* Text alignment: right

If the site is in English mode:

* Display ONLY English content
* Display:

  1. Terms of Use – Israel (English)
  2. Immediately after: Terms of Use – Europe (English)
* Layout direction: LTR
* Text alignment: left

Do NOT mix languages on the same page.
Do NOT show Hebrew and English together.

CONTENT STRUCTURE:

Page should include:

1. Main Title:
   Hebrew:
   "תנאי שימוש"
   English:
   "Terms of Use"

2. Section 1:

* Israel version

3. Divider (visual spacing or subtle line)

4. Section 2:

* Europe / EU / GDPR version

Both sections should appear one after the other in the same page.

CONTENT SOURCE:

Use the exact legal texts provided earlier.

Store content in a structured way, for example:

legalContent = {
he: {
israel: "...",
europe: "..."
},
en: {
israel: "...",
europe: "..."
}
}

At runtime:

* detect current site language
* render matching content only

PAGE DESIGN REQUIREMENTS:

* Clean white background
* Premium minimalist design
* Comfortable reading width (max-width ~700–900px)
* Centered content container
* Generous vertical spacing between paragraphs
* No aggressive styling
* No warning icons
* No red colors

TYPOGRAPHY:

* Body font size: 14px–16px
* Line height: 1.6–1.8
* Paragraph spacing: consistent and readable
* Headings slightly larger but not aggressive

SECTION TITLES:

Hebrew:
"תנאי שימוש – ישראל"
"תנאי שימוש – אירופה"

English:
"Terms of Use – Israel"
"Terms of Use – Europe"

DIRECTION HANDLING:

If Hebrew:

* dir="rtl"
* align text right
* ensure punctuation behaves correctly

If English:

* dir="ltr"
* align text left

NAVIGATION / ACCESS:

* Add link in footer: "תנאי שימוש" / "Terms of Use"
* Optionally add link in main menu or legal section

SEO / META:

* Page title:
  Hebrew: "תנאי שימוש"
  English: "Terms of Use"

* URL suggestion:
  /terms (single URL with dynamic language)
  OR
  /he/terms and /en/terms if routing supports it

TECH IMPLEMENTATION:

* Use a reusable component, e.g.: <TermsPage />

* Detect language from:
  global state / router / i18n provider

* Render content conditionally

* Avoid duplication in code

* Ensure fast load (static content preferred)

RESPONSIVENESS:

Desktop:

* centered layout
* readable margins

Mobile:

* full width with padding
* font readable
* no horizontal scrolling

ACCESSIBILITY:

* proper heading hierarchy (h1, h2)
* readable contrast
* selectable text
* screen reader friendly

FINAL REQUIREMENTS:

* The page must be static, readable, and easy to scroll
* No popups inside this page
* No collapsing sections (keep all text visible)
* Content must be long, continuous, and formal
* Must reflect full legal text as provided

FINAL RESULT:

When user switches language:
→ the entire Terms of Use page switches language instantly
→ no mixing
→ no reload issues
→ correct RTL/LTR behavior

END.

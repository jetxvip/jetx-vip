JetX.vp — FINAL REQUIREMENT BEFORE PRODUCTION (NO EXCEPTIONS)

This is a strict requirement before deployment.

Everything that currently works in development MUST work identically in production.

There is zero tolerance for “works in dev but not in production”.

---

## CORE REQUIREMENT

The system must behave as a fully synchronized, production-grade system.

No feature is allowed to break, degrade, or behave differently after deployment.

---

## 1. FULL DATA SYNCHRONIZATION (CRITICAL)

All data must be:

* persistent
* shared across devices
* consistent in real-time or near real-time

This includes:

### Leads

* any lead submitted from ANY device must appear in admin
* must include full data
* must persist after refresh
* must be visible on all devices

### Contact Data

* phone
* email
* WhatsApp

Changes in admin MUST:

* update the live site
* reflect across ALL devices
* appear in ALL components (header, footer, CTA, etc.)

### Offers / Deals

* when a deal is added or edited → must appear on live site
* must persist
* must be visible across devices

### Empty Legs Flights

* when added/updated → must appear on site
* must be synced globally
* must not be device-dependent

---

## 2. ADMIN PANEL MUST BE SOURCE OF TRUTH

The admin panel is NOT a local tool.

It is the single source of truth for the entire system.

Any change made in admin MUST:

* be saved in persistent storage
* reflect in production immediately or on next fetch
* be visible across all devices and sessions

NO local-only behavior is acceptable.

---

## 3. PRODUCTION ENVIRONMENT MUST MATCH DEV BEHAVIOR

Everything currently working in Orchids MUST work the same in production:

* login / auth
* session timeout
* leads system
* notes system
* notifications (bell + sound)
* contact sync
* offers / flights management

No missing features.
No partial functionality.
No “only works locally”.

---

## 4. NO LOCAL STORAGE DEPENDENCY

LocalStorage may be used only for temporary UX.

BUT:

The system must NOT rely on localStorage for core data.

All critical data must come from:

* KV
* API
* persistent backend

If localStorage is cleared → system must still work correctly.

---

## 5. CROSS-DEVICE TEST REQUIREMENT

You must ensure:

* admin on desktop
* user submits lead from mobile
* lead appears in admin
* refresh → still there
* open from another device → still there

Same applies to:

* contact updates
* deals
* flights
* notes

---

## 6. DEPLOYMENT READINESS (MANDATORY)

Before final delivery:

* remove all dev-only files:

  * .env.local
  * dev-api-server
  * Vite proxy

* ensure Vercel environment variables are used correctly:

  * ADMIN_EMAIL
  * ADMIN_PASSWORD
  * JWT_SECRET
  * KV credentials

* ensure API routes work in serverless environment (not only dev server)

---

## 7. FAILURE CONDITIONS (NOT ACCEPTABLE)

The following are NOT acceptable in production:

* data visible only on one device
* leads not appearing in admin
* contact changes not updating site
* features working only after refresh
* notifications not triggering
* dev-only behavior leaking into production

---

## FINAL EXPECTATION

After deployment:

I must be able to:

1. Add a deal → see it live everywhere
2. Add an empty leg → see it live everywhere
3. Change phone/email → update everywhere
4. Receive a lead from any device → see it in admin
5. Add notes → persist and sync
6. Get notifications (bell + sound)
7. Refresh / switch device → everything still works

---

This must behave as a real production system, not a local demo.

Everything must work. No exceptions.

ORCHID IMPLEMENTATION PROMPT – COOKIE BANNER (SMART + GDPR + HIGH CONVERSION)

Build a cookie consent banner that is legally compliant (Israel + GDPR level) but designed to be minimal, clean, and not harm conversion rates.

==================================================
LANGUAGE LOGIC (CRITICAL)
=========================

The website supports two modes:

1. Hebrew
2. English

If Hebrew mode:

* show Hebrew banner
* RTL layout

If English mode:

* show English banner
* LTR layout

Do NOT mix languages.

==================================================
BANNER DESIGN (VERY IMPORTANT)
==============================

GOAL:

* minimal
* elegant
* non-intrusive
* premium look

POSITION:

* bottom of screen (full width bar) OR small floating bottom box
* must NOT block CTA buttons or forms
* must NOT feel like a popup

STYLE:

* soft background (white / light gray / brand color)
* subtle shadow
* rounded corners (if box)
* clean typography
* no aggressive colors
* no warning icons

==================================================
FUNCTIONAL REQUIREMENTS
=======================

1. CONSENT STORAGE

* store consent in localStorage or cookies
* remember user choice
* do NOT show banner again after acceptance

2. DEFAULT BEHAVIOR

* Necessary cookies: always active
* Analytics & Marketing: optional

3. GDPR-STYLE LOGIC (IMPORTANT)
   Before user clicks "Accept":

* do NOT fire marketing cookies
* do NOT fire tracking pixels (Meta / Google if possible)

After acceptance:

* allow all cookies

4. SETTINGS SAVE

* user can choose preferences
* store them
* apply accordingly

==================================================
UX RULES (CRITICAL)
===================

* banner must not block forms
* must not interrupt flow
* must not feel scary
* must be easy to dismiss
* must not appear repeatedly

JetX.vp — Critical Production Fixes (Auth, Admin, Leads, Data Persistence)

This is a critical production-level fix request.

The current system is not behaving as a real admin-controlled production system.
There are fundamental issues in authentication, data persistence, lead handling, and admin authority over the live site.

These must be fixed as a unified system, not as isolated patches.

---

## 1. AUTHENTICATION & SESSION MANAGEMENT

### Issues:

* Login fields are pre-filled / not empty
* Credentials appear inside inputs
* Session never expires
* User stays logged in indefinitely unless manually logging out

### Requirements:

#### 1.1 Empty Login Fields

* Email and password inputs must ALWAYS be empty
* No default values
* No example credentials
* No autofill from localStorage or state

#### 1.2 No Stored Credentials

* Do NOT store email/password anywhere in:

  * localStorage
  * sessionStorage
  * frontend state persistence
* Only store a secure token if needed

#### 1.3 Session Expiration (10 minutes inactivity)

* Implement inactivity-based timeout (10 minutes)
* Reset timer on:

  * mousemove
  * click
  * scroll
  * keydown
  * touch events
* On timeout:

  * clear auth token
  * reset auth state
  * redirect to `/login`
  * show message: "Session expired due to inactivity"

#### 1.4 Token Validation

* `/admin` must always validate token
* Expired or invalid token → force logout
* No fallback or bypass allowed

#### 1.5 Logout Behavior

* Logout must:

  * clear all auth state
  * clear token
  * redirect to `/login`
  * prevent access via back navigation

## 3. LEAD CAPTURE SYSTEM (CRITICAL FAILURE)

### Current problem:

Users submit forms but NOTHING reaches the admin.

This is unacceptable.

### Requirements:

#### 3.1 Unified Lead System

All forms must feed into ONE centralized system.

Sources include:

* Quote request form
* Empty Legs interest
* Special Offers interest
* Any modal / CTA form

#### 3.2 Every submission must:

* Be saved
* Appear in admin
* Be identifiable by type/source
* Include full context of what user requested

#### 3.3 Required Lead Structure:

Each lead must include:

* full name
* phone
* email
* message
* inquiry type
* source (page/form)
* timestamp
* status (new / viewed / contacted / closed)

Additional metadata:

Empty Leg:

* flight / route
* identifiers if exist

Special Offer:

* offer title
* offer ID

Quote:

* route
* passengers
* travel details

#### 3.4 Admin Panel — Leads Section

Add a clear section:

* list of leads
* newest first
* full details
* status management

#### 3.5 Real-Time Notification

When a new lead arrives:

* visual alert (badge/highlight)
* sound notification (chime)

If admin panel is open → must immediately indicate new lead.

#### 3.6 Persistence (CRITICAL)

* NOT localStorage
* MUST be real persistent storage
* Data must:

  * survive refresh
  * be visible across devices
  * work in production

If needed → use Vercel KV or proper backend storage.

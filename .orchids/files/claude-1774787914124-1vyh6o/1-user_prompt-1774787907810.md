Critical production issue — site crashing after API security changes

The site is currently broken in production (white screen on multiple pages).

After investigating, the issue is clear:

The public website is trying to read data from:

* /api/store?key=jetx_company
* /api/store?key=jetx_flights
* /api/store?key=jetx_offers
* /api/store?key=jetx_fleet
  (and others)

However, /api/store now requires JWT authorization for ALL requests, including GET.

As a result:

* Public requests return 401 Unauthorized
* The frontend receives invalid data (not arrays)
* The code then calls `.filter()` on invalid data
* This crashes the app → white screen

Console errors confirm:

* Multiple 401 responses from /api/store
* TypeError: filter is not a function

Required fixes:

1. Public read access must be restored
   Allow unauthenticated GET requests for public keys only:

* jetx_company
* jetx_flights
* jetx_offers
* jetx_fleet

Keep JWT protection for:

* all writes (POST/PUT/DELETE)
* sensitive data (leads, clients, subscribers)

OR create a separate public read-only endpoint.

2. Add frontend safety
   Every `.filter`, `.map`, `.length` must be protected:

* Array.isArray(data) ? data : []

No page should crash due to invalid/missing data.

3. Ensure admin saves valid structures
   Prevent malformed data from being stored in KV.

This is a blocking production issue and must be fixed immediately.

# JetX.vip — Global Audio Experience Feature

## Goal

Add an optional global audio experience to the website.

The audio must:

* be configured from the admin panel
* play across the entire site
* continue playing while the user navigates between pages
* be user-initiated so browsers do not block it
* be controllable from the floating action buttons

Do NOT change the current design except for adding this feature.

---

## User Flow

1. User enters the site.
2. Existing intro animation plays.
3. After the intro completes, show a centered modal asking whether to enable the audio experience.
4. User can choose:

   * Enable sound
   * Continue without sound
5. If user enables sound:

   * audio starts immediately
   * audio continues across all routes/pages
   * floating speaker button controls mute/unmute
6. If user declines:

   * do not play audio
   * floating speaker button can still allow enabling later if appropriate

---

## Browser Autoplay Requirement

Audio must NOT autoplay before user interaction.

The first audio playback must be triggered directly by a user click/tap.

This is required so production browsers do not block playback.

---

## Admin Requirement

In the admin panel, add a new setting:

### Audio Experience URL

Admin should be able to enter/update:

* direct audio URL, preferably `.mp3`
* optionally YouTube URL only if implemented safely

Recommended:
Use direct audio files first for production reliability.

Store this setting in the existing admin/company/settings persistence system if available.

Do not create a fragile one-off local-only setting.

---

## Technical Requirement

Create a global audio provider/player mounted at the app/provider level, not inside individual pages.

Audio state must survive client-side navigation.

It should track:

* audioUrl
* hasUserChosen
* isPlaying
* isMuted
* volume
* whether prompt has been dismissed

Do not remount/reset audio when navigating between routes.

---

## Modal Design

Create a modal visually similar to the reference:

* blurred/dimmed page background
* centered circular luxury-style card
* speaker/equalizer icon at top
* short Hebrew copy by default, with multilingual support if possible
* primary button:
  Hebrew: הפעל סאונד
  English: Enable Sound
  Arabic: تفعيل الصوت
  Russian: Включить звук
* secondary text button:
  Hebrew: המשך ללא סאונד
  English: Continue without sound
  Arabic: المتابعة بدون صوت
  Russian: Продолжить без звука

The modal must appear only after the intro animation is complete.

---

## Floating Speaker Button

Add a speaker button to the existing floating buttons.

It should:

* match the existing floating button style
* use a speaker icon
* indicate muted/unmuted state
* allow toggle mute/unmute
* remain visible across pages
* not break WhatsApp/email/user floating buttons

Behavior:

* if audio was enabled, button toggles mute/unmute
* if audio was declined or not started, button may open the audio prompt or start audio after click

## Persistence

Use session-level behavior:

* Do not show the modal repeatedly on every route change.
* Once the user chooses, remember it for the current session.
* On a full new visit/session, it may ask again.
* Do not use permanent localStorage unless necessary.

---

## YouTube URL Handling

If YouTube support is added:

* do not autoplay before user click
* use a safe embedded player approach
* ensure it continues across client-side navigation
* handle invalid URLs gracefully

If YouTube support is too fragile, implement direct audio URL first and clearly report that YouTube should be converted to a direct MP3/audio URL for best production reliability.

---

## Production Safety

Verify:

* audio starts only after user click
* audio continues across internal navigation
* mute/unmute works
* no hydration errors
* no console runtime errors
* no layout shift
* public design remains unchanged
* admin setting persists

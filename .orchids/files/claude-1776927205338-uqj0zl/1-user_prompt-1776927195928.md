## 2) Wrong language flashes on refresh

On refresh, the page first renders in English and only then switches to Hebrew.

This is incorrect UX.

### Required behavior:

The correct language should render immediately on first visible paint, without flashing English first.

### Likely cause:

Language state is being initialized to `en` and then corrected in `useEffect`, which causes a visible language flash after hydration.

### Required fix:

Adjust the language initialization so that:

* hydration mismatch is still avoided
* but the wrong language does not visibly flash before switching

Do this safely.

---

## 3) Intro behavior is now over-fixed

Previously, the intro replayed too often.
Now it does NOT replay even on full refresh.

This is also incorrect.

### Required behavior:

* intro should NOT replay on internal navigation
* intro SHOULD replay on full browser refresh / fresh page load

### Required fix:

Adjust intro persistence logic so that it survives internal route changes only, but does not persist across full refresh in a way that suppresses the intro entirely.

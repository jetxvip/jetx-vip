IMPORTANT — ALL NEW FEATURES MUST BE CONNECTED TO THE EXISTING ADMIN PANEL

Do not create any hardcoded/static version of these new features.

ADMIN CONNECTION REQUIREMENTS:

1) HERO BOOKING / SEARCH BAR
The new booking/search bar on the homepage must be connected to the admin lead system.

When a visitor submits:
- From
- To
- Date
- Passengers
- Optional message (if added)

It must be saved as a lead inside the admin panel.

This lead should appear in the admin leads section together with the other incoming requests.

2) WHATSAPP FLOATING BUTTON
The floating WhatsApp button must NOT use a hardcoded phone number.

It must automatically use the WhatsApp number/link defined in the admin panel contact settings.

If the WhatsApp number is changed in admin, the floating button must update automatically across the whole site.

3) CONTACT / COMPANY DETAILS
Any phone number, email, WhatsApp link, or other contact detail shown in these new sections must always come from the admin panel settings.

Do NOT hardcode them in the frontend.

4) ADMIN-DRIVEN CONTENT
Any configurable text, CTA destination, or contact channel added in these new homepage sections should be editable from admin whenever possible.

5) DATA PERSISTENCE
All submitted leads and all admin settings must persist and remain saved after refresh.

Goal:
Everything added now must work as part of the real admin system, not as static demo UI.

## 5. Security / Secrets

Very important:

Admin username/password must NOT be hardcoded in the codebase.

Check the project for any hardcoded secrets, including:

* admin username
* admin password
* JWT secret
* KV tokens
* API tokens
* private keys
* hidden admin path if treated as sensitive

### Required production approach

Use Vercel Environment Variables.

Expected env vars should include, if used by the project:

```txt
ADMIN_EMAIL
ADMIN_PASSWORD
JWT_SECRET
KV_REST_API_URL
KV_REST_API_TOKEN
```

If the project uses different names, list the exact names.

### Required checks

* `.env` files must NOT be committed to GitHub
* `.env.local` must be in `.gitignore`
* no real credentials in source files
* no real credentials in frontend-exposed code
* no `NEXT_PUBLIC_` prefix for private secrets
* admin authentication must be verified server-side
* protected admin data must not be publicly accessible

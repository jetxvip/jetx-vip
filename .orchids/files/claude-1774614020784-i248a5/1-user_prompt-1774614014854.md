JetX.vp — Admin Panel Fix (Text Visibility + Notes CRM Behavior)

The system is mostly working, but two critical issues must be fixed to make the admin usable in real workflow:

---

## 1. TEXT VISIBILITY (CRITICAL)

### Problem:

Some text in the admin panel is not readable.
Background is dark/brown and text blends into it.

### Requirement:

All text must always be clearly visible.

Fix:

* ensure strong contrast between text and background
* no low-contrast combinations
* text must be readable in all admin components

This applies especially to:

* lead cards
* notes section
* any text inside lead details

Unreadable text is not acceptable in a production admin.

---

## 2. NOTES SYSTEM — MUST BE TRUE CRM HISTORY

### Problem:

Currently:

* I can type a note
* but it does NOT get saved as a history under the lead
* it stays in the input instead of becoming a persistent timeline

### Required behavior:

Each lead must support a running notes history, like a CRM.

---

### 2.1 Add Note Flow

* user types a note
* clicks "Add Note"
* note is saved
* input field is cleared

---

### 2.2 Notes History Display

All notes must appear under the input box as a list.

Each note must include:

* note text
* timestamp (date + time)

Example:

[12:00] Client asked to be called at 14:00
[14:05] Client said he wants to close the flight in the evening

---

### 2.3 Multiple Notes

* must support multiple notes per lead
* each new note is appended below previous ones
* notes are ordered chronologically

---

### 2.4 Persistence (CRITICAL)

Notes must:

* be saved in persistent storage (not just local state)
* survive refresh
* be visible from another device
* remain attached to the lead until it is deleted

---

### 2.5 Correct Behavior

* adding a note does NOT overwrite previous notes
* input is cleared after saving
* no duplication
* no disappearing notes

---

## 3. IMPORTANT

Do NOT change:

* lead statuses (they already work correctly)
* lead flow
* admin structure

Only fix:

* text visibility
* notes behavior

---

## 4. EXPECTED RESULT

After fix:

* all admin text is clearly readable
* I can:

  * open a lead
  * change status (e.g. "on progress")
  * add multiple notes over time
  * see all notes listed under the lead
  * track full communication history
* notes remain after refresh and across devices
* notes are only removed when the lead is deleted

---

This is required for real lead management workflow (CRM-like behavior).

Final reliability improvement for Hero YouTube video.

Current behavior is almost perfect, but there is one edge case:

Right now, iframe onLoad cancels the 10s fallback timer.
However, onLoad only means the iframe document loaded — NOT that the YouTube player is actually working.

This can cause a broken state where:
- iframe loads (onLoad fires)
- timer is cancelled
- but the player never becomes ready or starts buffering/playing
- result: stuck blank/black video with NO fallback

Required adjustment:

1. Do NOT treat iframe onLoad as a valid "video signal".

2. The 10s fallback timer should ONLY be cleared by:
   - YouTube onReady
   - onStateChange = BUFFERING (3)
   - onStateChange = PLAYING (1)

3. iframe onLoad can still be used for:
   - setting iframeLoaded (for opacity / visibility)
   - but MUST NOT cancel the fallback timer

4. Keep gotAnySignalRef, but only set it to true when:
   - onReady fires
   - OR onStateChange BUFFERING/PLAYING fires

5. If iframe loads but YouTube player never initializes,
   the 10s timer should still trigger fallback.

6. Keep existing behavior:
   - onError (100/101/150) → immediate videoFailed
   - fallback image behind iframe (z-index layering unchanged)
   - no retries / no remounts

Goal:
We only trust real YouTube player signals — not iframe load —
so we avoid both:
- false fallback (too early)
- AND stuck broken video with no fallback

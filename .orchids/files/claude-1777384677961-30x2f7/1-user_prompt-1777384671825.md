Hydration error in Navbar due to inconsistent inline styles between server and client.

The mismatch shows:
- Server: bottom: "84px"
- Client: bottom: 84

This causes:
Hydration failed because the server rendered HTML doesn't match the client.

Fix required:

1. Ensure inline style values are consistent between SSR and client:
   - Always use strings with units for CSS values:
     bottom: "84px"
   - Never mix number (84) and string ("84px")

2. Find any dynamic calculation like:
   bottom: someValue

   and normalize it:
   bottom: `${someValue}px`

3. If the value depends on window (viewport, scroll, etc):
   - Do NOT calculate it during SSR
   - Move it into useEffect or useState after mount

Example:
const [bottom, setBottom] = useState(84);

useEffect(() => {
  setBottom(84); // or calculated value
}, []);

style={{ bottom: `${bottom}px` }}

4. Avoid:
   typeof window !== 'undefined' inside render
   Date.now()
   Math.random()
   or any value that differs between server and client

5. If needed, temporarily wrap the Navbar in:
   suppressHydrationWarning
but prefer fixing the root cause instead.

Goal:
Server and client must render identical HTML/styles on first render.

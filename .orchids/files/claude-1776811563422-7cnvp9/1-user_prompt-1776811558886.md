# Fix Intro Behavior — Show Only on First Entry, Not on Internal Navigation

The homepage intro is currently re-playing every time I navigate back to Home.

This is incorrect.

## Required behavior

The intro must appear only when:

* the user enters the site for the first time
* or the page is fully refreshed / newly opened

The intro must NOT appear again during internal navigation, for example:

* Home → Services → Home
* Home → Empty Legs → Home
* Home → About → Home

---

## Root issue

The current intro state is being reset on internal route navigation.

That means the intro visibility is tied to component remounting instead of session-level visit state.

---

## Required fix

Update the intro logic so that it persists across internal navigation.

Preferred approach:

* use `sessionStorage` (recommended)
  or
* another safe app-level persistence approach

### Expected logic

* on first site entry, intro plays
* once completed, mark intro as seen for the current session
* on internal route changes, do not replay intro
* if the user refreshes the page, behavior should match the intended session rule

Do NOT change the visual design of the intro.
Do NOT change timing or styling unless necessary.
Only fix when it appears.

# JetX.vip — Mandatory Visual Restoration (Exact Pre-Migration Design)

The current Next.js / SEO migration is not visually acceptable.

The site must look **exactly like it looked before the migration**.

This is now the top priority.

## Critical requirement

I do **not** want:

* a refreshed version
* a cleaned-up version
* an approximation
* a reinterpretation
* a “close enough” version

I want the site to look **exactly** like the pre-migration design.

The architecture improvements may stay, but the visual result must be restored to the original design with pixel-level fidelity.

---

## What this means

Restore the original:

* layout
* spacing
* alignment
* typography
* font sizing
* section proportions
* card sizes
* button sizes and styling
* paddings / margins
* navigation appearance
* hero appearance
* footer appearance
* page flow
* responsive behavior
* visual hierarchy
* animation behavior if it existed before
* language switcher appearance

Do NOT redesign anything.

Do NOT normalize spacing.

Do NOT substitute styles.

Do NOT simplify the UI.

Do NOT make subjective design decisions.

---

## Required workflow

### 1. Compare against the pre-migration version

Use the pre-migration state / commit / snapshot as the visual source of truth.

The pre-migration design is the only approved design reference.

### 2. Restore visual parity

Restore the frontend so that the rendered site visually matches the original pre-migration version as closely as possible.

### 3. Keep the SEO architecture underneath

Keep:

* Next.js architecture
* SSR / SSG improvements
* metadata
* sitemap / robots
* structured data
* route improvements

But all of that must remain invisible from a design perspective.

### 4. Change only what is required for visual restoration

If the current design drift came from:

* wrappers
* provider order
* layout structure
* class application order
* container differences
* hydration-side rendering differences
* global CSS loading differences
* font loading behavior
* Link/render differences
* app layout wrappers

then fix those causes — but do not alter the intended original design.

---

## Required final deliverable

When done, provide:

1. exact list of files changed
2. exact cause of the visual mismatch
3. confirmation that the original pre-migration design has been restored
4. confirmation that SEO architecture is still intact
5. any remaining differences, if any exist at all

If any difference remains, list it explicitly.

---

## Important instruction

If necessary, temporarily revert the visual layer to the pre-migration implementation and then re-apply the SEO architecture more carefully underneath it.

The final result must be:

* pre-migration design
* post-migration SEO architecture

No compromise on visual fidelity.

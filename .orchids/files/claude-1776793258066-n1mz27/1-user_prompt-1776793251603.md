# JetX.vip — Multilingual SEO Expansion (40 SEO Pages × 4 Languages = 160 Indexable Pages)

## Objective

Build a multilingual SEO landing-page system for JetX.vip that creates **40 secondary SEO pages** across **4 languages**:

* English (`en`)
* Hebrew (`he`)
* Arabic (`ar`)
* Russian (`ru`)

Total pages to generate and publish: **160**

The goal is to significantly expand JetX.vip’s search visibility for high-intent private aviation queries, while keeping the site visually identical to the current luxury design.

---

## Non-Negotiable Rules

### Do NOT:

* redesign the site
* change visual hierarchy
* change spacing, typography, cards, buttons, or premium aesthetic
* add these SEO pages to the main navigation
* add clutter to the UI
* create blog-style spam content
* use thin/duplicate content
* use keyword stuffing
* use cheap or generic language

### MUST:

* preserve the exact current design system
* use the current site’s visual components/layout
* create these as **secondary SEO pages**
* make them indexable
* include them in sitemap
* generate proper multilingual metadata
* implement canonical + hreflang properly
* keep tone premium, elegant, high-net-worth, aviation-native

---

## Architecture / Routing Requirements

Create the SEO pages as real SSR/SSG pages in Next.js.

Preferred route structure:

```txt
/en/private-jet-london
/he/private-jet-london
/ar/private-jet-london
/ru/private-jet-london
```

Same pattern for all 40 slugs.

### Requirements:

* each page must be a real route
* each page must render meaningful HTML on first load
* each page must have language-specific metadata
* each page must have hreflang links for all 4 versions
* canonical must point to the page’s own language URL
* all pages must appear in sitemap.xml
* pages should NOT appear in the main nav or visually dominant menus

---

## Implementation Strategy

Build this as a reusable SEO page system with:

* one shared layout/template
* page-type-specific content generation
* per-language content variations
* page data stored in a structured config/content source

Use the existing JetX luxury design and sections.
Do NOT invent a new page style.

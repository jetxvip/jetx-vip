# Fix Default Language Detection — Israel Must Load Hebrew First

Current issue:
The site still loads English first, even for users in Israel.

Expected behavior:

* Users from Israel should load Hebrew (`he`) by default
* Users outside Israel should load English (`en`) by default
* If the user manually selected a language before, respect that explicit choice

## Important

Do NOT rely only on browser language or timezone.

Browser language can be English even when the user is physically in Israel.

## Required production logic

Use Vercel / request country detection if available:

* If request country is `IL` → default language = `he`
* Otherwise → default language = `en`

Use headers such as:

* `x-vercel-ip-country`
* or the correct Vercel geolocation/header method available in this project

## Required priority order

1. Explicit user-selected language from localStorage/cookie
2. Vercel country/IP detection:

   * `IL` → `he`
   * otherwise → `en`
3. Browser locale fallback:

   * Hebrew locale → `he`
   * Arabic locale → `ar`
   * Russian locale → `ru`
4. Final fallback → `en`

## No flicker requirement

The page should not visibly load English first and then switch to Hebrew.

If needed, set a language cookie/server-derived initial language so the first render already knows the correct default.

## Orchids/dev note

If Orchids/dev cannot provide real Vercel country headers, report that clearly.

But production on Vercel must work:

* Israel IP → Hebrew first render
* Non-Israel IP → English first render

## Do not change

* design
* SEO pages
* admin
* audio feature
* language switcher UI

6. confirmation: outside Israel → English first

Fix the homepage Hero YouTube background video.

Current issue:
The hero video briefly appears for about one second and then falls back to the fallback image. This must not happen. The fallback image should only appear if there is no valid YouTube URL/video ID, or if YouTube truly fails to load after a reasonable timeout. Do NOT switch to fallback just because autoplay is delayed.

Files to inspect and update:
- src/page-components/Home.jsx
- src/index.css
- any related admin/company config that stores company.heroVideoUrl

Requirements:
1. In the Hero component, extract the YouTube video ID from company.heroVideoUrl using a robust helper that supports:
   - youtube.com/watch?v=
   - youtu.be/
   - youtube.com/embed/
   - youtube.com/shorts/
2. If a valid YouTube ID exists, always render the YouTube iframe as the primary hero background.
3. Use a proper YouTube embed URL:
   https://www.youtube.com/embed/{VIDEO_ID}?autoplay=1&mute=1&muted=1&loop=1&playlist={VIDEO_ID}&controls=0&modestbranding=1&playsinline=1&rel=0&enablejsapi=1&disablekb=1&fs=0&iv_load_policy=3&origin={window.location.origin}
4. The iframe must be:
   - absolute
   - centered
   - cover the full hero area like background-size: cover
   - pointer-events: none
   - z-index above the fallback image
   - opacity 1 once loaded
5. The fallback image must stay behind the iframe and must NOT cover the video.
6. Remove any logic that switches to fallback too aggressively.
7. Only show fallback if:
   - there is no valid YouTube ID
   - or iframe loading fails completely
   - or after at least 8–10 seconds there is still no iframe load event
8. Do not use a 1-second or 2-second fallback timeout.
9. Add state:
   - iframeLoaded
   - videoFailed
10. On iframe onLoad, set iframeLoaded true and keep showing the iframe.
11. Do not hide/remove the iframe after it loads.
12. Keep the fallback image visible underneath while iframe is loading, but once iframe loads it should visually dominate.
13. Make sure this works on desktop, mobile, Safari, Chrome, Firefox, iOS and Android as much as browser autoplay policies allow.
14. Important: YouTube autoplay can only reliably work muted, so keep the video muted.
15. Do not add custom play buttons or user interaction requirements.
16. Preserve all existing hero text, layout, gradients, buttons, responsiveness and design.

Implementation guidance:
Replace the current raw iframe/fallback block in Hero with a controlled structure like:

- fallback image div: z-index 0
- youtube iframe: z-index 1, shown when valid ID and not videoFailed
- overlays/gradients: z-index 2
- content: z-index 10

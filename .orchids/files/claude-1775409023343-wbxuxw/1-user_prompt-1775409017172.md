אני צריך להקשיח את הגישה למסך האדמין באתר שלי ולהוסיף שכבת אבטחה נוספת ברמת ה-routing.

### המצב הנוכחי:

כרגע ניתן לגשת למסך האדמין דרך נתיבים כמו:

* /admin
* /login

וזה לא רצוי.

### הדרישה:

אני רוצה שהגישה למסך האדמין תתאפשר **אך ורק** דרך נתיב ייחודי וסודי:

* /adminitojetxvip

### כללים ברורים:

1. כל ניסיון גישה ל:

   * /admin
   * /login
   * /dashboard
   * או כל נתיב סטנדרטי אחר שקשור לאדמין
     **לא יוביל למסך האדמין בשום צורה**

2. עבור הנתיבים הללו:

   * אפשר להחזיר 404
   * או לבצע redirect לעמוד הבית
   * או להציג דף “Not authorized”

3. רק כאשר המשתמש נכנס ל:

   * /adminitojetxvip
     המערכת צריכה לטעון את מסך האדמין

4. חשוב:

   * הנתיב הסודי לא צריך להיות גלוי בקוד ציבורי (אם אפשר – להגדיר אותו דרך environment/config)
   * לא לבצע redirect אוטומטי מנתיבים אחרים לנתיב הסודי
   * לוודא שאין לינקים פנימיים שמובילים אליו

### אבטחה נוספת (אם אפשר):

* להוסיף בדיקת authentication (JWT / session / token)
* להוסיף role-based access (רק admin)
* לשקול rate limiting על הנתיב
* להוסיף middleware שבודק הרשאות לפני טעינת האדמין

### מטרה:

למנוע לחלוטין גישה למסך האדמין דרך נתיבים צפויים/סטנדרטיים, ולאפשר גישה רק דרך נתיב ייחודי ולא טריוויאלי.

תבצע את זה בצורה נקייה, מאובטחת, וסקיילבילית.

# Bug — Audio Plays Twice After iOS Fix

After the latest iOS Safari audio fix, the music now plays twice at the same time on Android.

This means there are likely two active players running simultaneously.

## Likely cause

The new pre-created YouTube player in `AudioContext.jsx` is playing, while the fallback `YouTubeAudioPlayer.jsx` also mounts/plays with autoplay.

## Required fix

Ensure only ONE audio player can ever play at a time.

Requirements:

1. If the pre-created YouTube player is ready and used:

   * do NOT mount/play the fallback `YouTubeAudioPlayer`
   * disable fallback autoplay

2. If fallback is used:

   * make sure the pre-created player is stopped or not playing

3. Add a single source-of-truth flag, for example:

   * `activePlayerType: 'preloaded' | 'fallback' | null`

4. Before starting playback, stop/pause any other player instance.

5. Confirm:

   * Android plays only once
   * iPhone/Safari still works
   * Desktop still works
   * mute/unmute controls the active player only

## Do NOT

* change design
* remove the audio modal
* remove the floating speaker button
* break MP3/direct audio playback

## Deliverable

Send:

1. root cause
2. files changed
3. how duplicate playback was prevented
4. confirmation Android plays once
5. confirmation iPhone/Safari still works
